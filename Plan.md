## Перечень автоматизируемых сценариев:
* покупка по сервису платежей с валидными данными карты
* покупка по кредитному сервису с валидными данными карты
* проверка полей формы невалидными данными: отправка пустой формы, неверные данные карты (номер, год, месяц, владелец)
* оформление репортов с Allure
* SQL-запросы для проверки внесения в данных в БД
## Перечень используемых инструментов с обоснованием выбора:
* Java 11 - стабильная и надежная версия
* Gradle - Понятный синтаксис, удобное встаивание зависимостей в проект, компактнее код, чем в Maven
* JUnit5 - Удобно писать и группировать тесты, есть аннотации для разных типов запуска
* Selenide - более компактный код, чем в работе с Selenium. Есть интеграция с Allure
* MySQL - стабильная и быстрая база данных. Легкая настройка
* Lombok - сокращения кода (геттеры, сеттеры, конструкторы) в проекте через аннотации
* Allure - удобный и интуитивно понятный интерфейс для составления отчетности без дополнительного расхода ресурсов локальной машиной по сравнению с ReportPortal
* Faker - генерация данных для тестирования. Помогает избежать использования одинаковых данных
* Apache DbUtils - простая библиотека для выполнения SQL-запросов и проверки данных в БД
* DBeaver - вспомогательный инструмент. Удобный графический интерфейса для работы с базами данных
## Перечень и описание возможных рисков при автоматизации:
* Изменение тестовых данных
