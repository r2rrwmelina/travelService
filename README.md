# Travel Service Test Automation

Автоматизация тестирования веб-сервиса покупки тура "Путешествие дня". Сервис предоставляет два способа оплаты: по дебетовой карте и в кредит.
Тесты покрывают позитивные и негативные сценарии покупки по дебетовой карте, а также валидацию полей формы оплаты.

## Начало работы

Скопировать удаленный репозиторий на компьютер и создать локальный. В терминале написать команду git clone и вставить ссылку:
```
git@github.com:r2rrwmelina/travelService.git
```

### Prerequisites

Что нужно установить на ПК для использования:

```
Java 11
Chrome browser
Docker Desktop
Git
```

### Установка и запуск

```
1. Запуск контейнера: docker compose up
2. Запуск учебного приложения: java -jar ./artifacts/aqa-shop.jar. Убедиться, что приложение запустилось на порту 8080
3. Запуск тестов: ./gradlew clean test
4. Запуск отчета в браузере: ./gradlew allureServe
```
